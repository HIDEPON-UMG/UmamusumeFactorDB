# Google Form 作成手順

投稿者が画像を送信すると自動で処理されるよう、Google Form を設定する。

## 1. Form 作成

1. [Google Form](https://forms.google.com) → 新規作成 → 空白のフォーム
2. タイトル：`ウマ娘 因子投稿フォーム`（任意）
3. フィールドを追加：

   **フィールド 1：投稿者Xハンドル**
   - 質問の種類：記述式（短文）
   - 質問文：`トレーナーID (例: 12345678901234)`
   - 回答必須：ON

   **フィールド 2：画像**
   - 質問の種類：**ファイルのアップロード**
     - ※この種類は Google アカウントへのログインを要求する
   - 質問文：`ウマ娘の継承因子画面スクリーンショット (縦方向スクロール結合済み)`
   - 許可するファイル形式：**画像のみ**
   - ファイル数の上限：1
   - 最大ファイルサイズ：10 MB
   - 回答必須：ON

## 2. 回答の保存先を設定

1. Form 編集画面 → 「回答」タブ → 右上のスプレッドシートアイコン
2. **既存のスプレッドシートを選択** → `UmamusumeFactorDB` の書き込み先スプレッドシートを選ぶ
3. 新しいシートが自動作成される（例：`フォームの回答 1`）

## 3. Apps Script トリガ設定

回答先のスプレッドシートを開き、拡張機能 → Apps Script：

1. `Code.gs` 最新版を貼り付け済みであることを確認
2. 左メニュー **「トリガー」**（時計アイコン）
3. 右下 **「トリガーを追加」**
4. 設定：
   - 実行する関数：`onFormSubmit`
   - 実行するデプロイ：`Head`
   - イベントのソース：`スプレッドシートから`
   - イベントの種類：**`フォーム送信時`**
   - 失敗通知：毎日 / 即時 お好みで
5. 保存 → 権限承認ダイアログでスプレッドシート・Drive・外部 URL アクセスを許可

## 4. 動作確認

1. Form の「プレビュー」（目のアイコン）から自分で投稿
   - Xハンドル：`@test`
   - 画像：サンプル画像 `tests/fixtures/sample_oguricap.png`
2. 投稿ボタンを押してから 15〜30 秒待つ（Cloud Run のコールドスタート含む）
3. スプレッドシートを確認：
   - `フォームの回答 1` タブ：最新行に `submission_id` と `status=processed` が付記
   - `factors_normalized` タブ：3 行（main/parent1/parent2）追加

## 5. 公開設定（任意）

- Form の「送信」ボタン → リンク取得 → 投稿者に共有
- 「回答の受付」を ON/OFF で一時停止可能

## トラブル

- **トリガが動かない**：Apps Script → 実行履歴 でエラーを確認
- **画像が取れない**：Form のファイルアップロード設定で「画像のみ」になっているか、サイズ上限に収まっているか
- **Cloud Run 500 エラー**：`gcloud run services logs read factor-processor --region=asia-northeast1` でログ確認
- **権限不足**：トリガー初回実行時に権限承認ダイアログが出るので全て許可する必要あり（Google アカウントのセキュリティ設定で「未検証のアプリ」を許可）
